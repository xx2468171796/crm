# Order Type Totals Specification

## Purpose

在财务工作台增加新单/复购分类统计，区分首次购买客户和回头客的业绩贡献。

## ADDED Requirements

### Requirement: 新单/复购分类统计

系统应在财务工作台汇总区域显示新单合计和复购合计。

#### Scenario: 显示新单合计
- **WHEN** 用户访问财务工作台
- **THEN** 显示"新单合计"统计卡片
- **AND** 新单定义为：客户在系统中按合同ID排序的第一个合同
- **AND** 金额为该类合同的已收金额总和

#### Scenario: 显示复购合计
- **WHEN** 用户访问财务工作台
- **THEN** 显示"复购合计"统计卡片
- **AND** 复购定义为：客户在系统中按合同ID排序的第二个及以后的合同
- **AND** 金额为该类合同的已收金额总和

#### Scenario: 筛选条件同步
- **WHEN** 用户应用任何筛选条件（时间、签约人、归属人等）
- **THEN** 新单/复购合计应同步更新，只统计符合筛选条件的合同

#### Scenario: 货币汇率转换
- **WHEN** 用户切换汇率模式（原始/固定/浮动）
- **THEN** 新单/复购合计应按选择的汇率模式转换显示
